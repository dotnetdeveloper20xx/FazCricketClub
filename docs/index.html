Architecture Highlights

Enterprise patterns and clean code principles ensuring maintainability and testability.

Clean Architecture

Layered solution with Domain, Application, Infrastructure, and Presentation tiers following SOLID principles

Repository Pattern

Abstracted data access layer enabling unit testing and database provider flexibility

Unit of Work

Transaction coordination across multiple repositories for data consistency

CQRS-Ready Services

Separated read/write operations with distinct DTOs for optimized data flow

Policy-Based Auth

Claim-based authorization with granular permission policies (View/Edit per feature)

Signal-Based State

Modern Angular signals for reactive, fine-grained state management without RxJS complexity

Soft Delete Pattern

Global query filters for logical deletion preserving data integrity and audit trails

FluentValidation

Declarative validation rules with custom messages and complex business logic support

Security Implementation

🔒

JWT Authentication

Symmetric key signing with issuer/audience validation and configurable expiry

🛡

Claim-Based Authorization

Granular permissions (Players_View, Teams_Edit, Admin_ManageUsers)

⏲

Rate Limiting

IP-based request throttling (200/min global, 10/min auth endpoints)

🔓

Security Headers

HSTS, CSP, X-Frame-Options, X-Content-Type-Options on all responses

📑

Correlation IDs

Request tracing with X-Correlation-Id for distributed debugging

🔐

Account Lockout

5 failed attempts triggers 5-minute lockout to prevent brute force
